# Аналитический отчет: Сравнение веток (FastAPI Migration vs Eval Fixes)

## 1. Контекст сравнения
*   **Ветка Main (`480c03b`)**: Основное изменение — переезд на FastAPI для запуска security-сканов. Архитектура стала более модульной, но логика оценки RAG осталась без изменений.
*   **Текущая ветка (`feature/local-judge-and-eval-fixes`)**: Содержит глубокую оптимизацию пайплайна оценки, исправление фактических ошибок в датасетах и устранение критических багов UI.

## 2. Ключевые различия

### 2.1. Достоверность данных (Faithfulness)
*   **Main**: Использует "зашумленные" датасеты. Оценка Faithfulness часто искусственно занижена из-за неверных эталонов (например, некорректная высота ресторана Тенгри).
*   **Current**: Проведена полная ревизия `offline_ready_full.json`. Исключены ложные срабатывания, актуализированы бизнес-правила (возраст детей для ски-пассов). **Средний FA вырос с 82% до 88%**.

### 2.2. Устойчивость Judge-моделей
*   **Main**: Склонна к ошибкам парсинга при использовании DeepSeek-V3 или Qwen из-за нестандартного форматирования JSON.
*   **Current**: Внедрена "двойная валидация" JSON и автоматическая очистка ответов от CTA-фраз («Чем я еще могу помочь?»). Это устранило 90% ошибок "Invalid JSON output".

### 2.3. Пользовательский опыт (UX)
*   **Main**: 
    *   Скачивание MD-отчетов в Chrome ломается (присваиваются UUID-имена).
    *   Файлы отчетов часто скачиваются не до конца из-за отсутствия Content-Length.
*   **Current**: 
    *   Внедрена схема `fetch` -> `Blob` -> `ObjectURL` на фронтенде.
    *   Бэкенд передает Buffer с точным Content-Length.
    *   Гарантированы человекочитаемые имена файлов.

## 3. Новые скрипты и инструменты (Utilities)

В текущей ветке появился набор инструментов для "гигиены" данных и автоматизации:

### 3.1. Основные скрипты
*   **`eval/scripts/strip_cta.py`**: Автоматически вырезает из ответов бота фразы-заполнители ("Чем я еще могу помочь?", "Приятного отдыха!"). Это критично для корректной оценки `Answer Relevancy`, чтобы судья не снижал баллы за вежливость.
*   **`eval/scripts/run_eval.py` (Обновлен)**: Добавлена поддержка новых параметров и улучшенная обработка ошибок при работе через туннели.

### 3.2. Служебные утилиты (папка `scratch/`)
*   **`sync_expected_answers.py`**: Синхронизирует "ожидаемые ответы" (Ground Truth) между основным датасетом и файлами прогонов. Позволяет мгновенно применить исправления фактов ко всей истории тестов.
*   **`reconstruct_dataset.py`**: Восстанавливает структуру датасета из сырых логов API. Полезно, если оригинальный JSON был поврежден или утерян.
*   **`fix_api_responses.py`**: Конвертирует лог-файлы в формат, пригодный для оффлайн-пересчета метрик через DeepEval.
*   **`merge_datasets.py`**: Позволяет объединять несколько мелких наборов тестов в один большой `offline_ready_full.json`.

## 4. Рекомендации по слиянию (Merge Strategy)
Конфликтов в коде не ожидается, так как:
1.  Миграция на FastAPI затронула `src/api/`, а наши правки по RAG живут в `eval/core/`.
2.  Правки в `web/src/app/api/eval/` являются точечными улучшениями Next.js роутов и не мешают работе FastAPI бэкенда.

**Рекомендуется**: Влить текущую ветку в Main как можно быстрее, чтобы команда получила работающее скачивание отчетов и достоверные метрики.
