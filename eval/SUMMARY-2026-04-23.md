# RAG Evaluation Pipeline Summary (Checkpoint)

## 1. Ядро оценки (eval/core)
*   **api_utils.py**: 
    *   Внедрена автоматическая очистка ответов бота от CTA («Чем я еще могу помочь?»).
    *   Логирование теперь сохраняет **все оригинальные колонки** датасета.
    *   Формат логов `api_responses.json` теперь на 100% совместим с оффлайн-прогоном (ключи `actual_output`, `retrieval_context` в виде списка строк).
*   **judges.py**:
    *   Включен режим `json_object` для всех судей.
    *   Добавлена «двойная броня» при парсинге JSON (валидация через roundtrip `json.loads` -> `json.dumps`).
    *   Таймауты увеличены до 300 сек для стабильности через туннели.
*   **eval_rag_metrics.py**:
    *   Исправлен баг с потерей логов (теперь `api_responses.json` генерируется всегда).

## 2. Данные (eval/datasets)
*   **20260422_deep_eval_quality_dataset.json**: Актуализированы эталоны (удалена неверная информация про Телецкое озеро, уточнены трассы и контакты).
*   **offline_ready_full.json**: Наш основной рабочий датасет для оффлайн-тестов, содержащий последние реальные ответы бота и чанки.

## 3. Протокол работы
*   **Онлайн-прогон**: Используется для получения свежих ответов от бота. Результат (`api_responses.json`) теперь сразу готов к оффлайн-пересчету.
*   **Оффлайн-прогон**: Используется для быстрой итерации по метрикам без лишних затрат на API бота.
